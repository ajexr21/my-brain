# AJSoft SecureVault PoC 범위 정의서 (Version 1.0)

## 🎯 프로젝트 개요: 규제 리스크 선제적 관리 시스템 구축
*   **목표:** 고객사가 현재 직면한 핵심 법규(예: GDPR, HIPAA 등 산업별 규정) 준수 관련 운영상의 사각지대와 비효율적인 감사 추적 프로세스를 발굴하고, 이를 AJSoft의 '불변 게이트웨이 기반 감사 추적 로깅'을 통해 최소화합니다.
*   **PoC 기간:** 4주 (총 1회 주간 보고서 제출)
*   **산출물(Deliverables):** 개선된 운영 프로세스 정의서, 규제 준수 위험도 점수 변화 데이터, 통합 관리자 대시보드 시연 자료.

## ✅ PoC 핵심 범위 (In Scope) - '위험 발견'에 집중합니다.
| 영역 | 기능/모듈 | 비즈니스 가치 (Pain Point 해소) | 테스트 목표 |
| :--- | :--- | :--- | :--- |
| **1. 감사 추적 로깅 엔진** | 모든 핵심 트랜잭션(로그인, 데이터 접근, 수정 등)의 발생 시점과 주체를 **불변성(Immutability)**을 보장하며 수집 및 기록합니다. | 💡 **책임 소재 명확화:** 누가, 언제, 무엇을 했는지 시간순으로 완벽하게 입증하여 내부 감사에 대비합니다. (가장 큰 고통점) | 핵심 트랜잭션 유형 5가지(CRUD 등)의 로그 데이터 수집 및 무결성 검증. |
| **2. 규제 위험 보고서 생성** | 수집된 로깅 데이터를 기반으로 '미준수 패턴'이나 '규정 위반 가능성이 높은 활동'을 자동 감지하고 시각화합니다. (예: 특정 시간대 비인가 접근 빈도 증가) | 🛡️ **사후 대응 → 사전 예방:** 단순 로그 나열이 아닌, 규제 관점의 위험도를 점수화하여 경영진에게 리스크 경고를 제공합니다. | 사전에 정의된 '규제 위반 시나리오' (3개 이상)에 대한 자동 탐지 보고서 생성 및 검증. |
| **3. 게이트웨이 연동 및 모니터링** | 기존 핵심 시스템(Legacy System)의 API 호출 진입점을 AJSoft 게이트웨이를 통해 거치도록 구현하고, 모든 트래픽을 감시합니다. | ⚙️ **전체 통제권 확보:** 사일로화된(Isolated) 내부 시스템 간의 데이터 흐름 전체를 한곳에서 모니터링할 수 있게 됩니다. | 실제 운영 환경의 API 요청 50건 이상을 게이트웨이를 통해 정상적으로 포착하고 로그 처리 확인. |

## ❌ PoC 제외 범위 (Out of Scope)
*   새로운 비즈니스 로직 개발 및 시스템 기능 추가 (MVP 이후 단계에서 진행).
*   사용자 인터페이스(UI/UX)의 전면 개편 또는 커스터마이징 요청.
*   PoC 기간 동안 발생한 규제 법규 변경에 따른 대응은 고객사 자체 프로세스 검토가 필요합니다.
